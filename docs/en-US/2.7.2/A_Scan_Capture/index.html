Ameliorations Open Bee Scan Capture

Restrict documents processed to extensions PDF / TIFF / TIF / PNG / DOC / DOCX

Only PDF / TIFF / TIF / PNG / DOC / DOCX extensions are authorized/allowed to be processed by Open Bee Scan Capture. All other extensions are automatically moved to the folder of the documents in error.

Ability to use the document processing date as metadata sent to Open Bee™ Portal

The date of document processing by Open Bee Scan Capture can now be used as metadata sent to Open Bee™ Portal.

Ability to add variables in the accounting export file path

It is now possible to use the following variables: "Folder Name", "Batch Identifier", "Export Date", "Batch Creation Date" or "Additional Folder Information Field" for the creation of the accounting export file path.

Ability to link indexes to analytical sections on equal values

It is now possible to assign an analytic section at 100% based on the value of an index if its value matches the name of the analytic section.

Management of the due date on Sage 100 Business Objects

If no payment condition is set to a third party, the expiry date of the entry will now correspond to the "Due Date" index in the case of using the Sage 100 Business Objects to export the entries.

Improvement of the rollback system on Sage 100 Business Objects

The rollback system in the case of using the Sage 100 Business Objects has been improved so as not to misuse the entire entry batch, but only the entries of the invoice in error.

Addition of the part number in the Sage 100 format export file (TXT file)

In order to avoid issues with part number overlap in Sage 100, the part number has been added in the case of a Sage 100 TXT format export.

Changing the behavior of the "part_number" variable in the export generator of accounting entries

Now the auto variable incremented "part_number" starts at 1 instead of the value of the document identifier in Open Bee Scan Capture.

Addition of pre-integrated tags in the export generator of accounting entries

-isVATLine : Allows to test if the entry line contains a VAT account
-isPersonalLine : Allows to test if the entry line contains a third-party account
-isClientLine : Allows to test if the entry line contains a customer account
-isSupplierLine : Allows to test if the entry line contains a supplier account
-isChargeLine : Allows to test whether the entry line contains a load account
