Correction Open Bee Scan Capture

Marker detection: malfunction when a marker is placed in the middle of a sentence

When a marker is placed in the middle of a sentence, a malfunction prevented detection of that marker. This problem is now fixed.

Markers not displayed in all customers interfaces

Markers may not be displayed in some cases. This problem is now fixed.

Installation problem of Open Bee Scan Capture on another drive than drive C:\

An installion problem of MongoDB causing a malfunction of Open Bee Scan Capture could occur in the case whereOpen Bee Scan Capture Server was installed on a different disk than the C:\ drive.

Scrollbar problem on the parameter table of the output to Open Bee™ Portal

A malfunction could occur with the scrollbar of the parameter table of the output to Open Bee™ Portal. This problem is now fixed.

Generic template used instead of the folder template when creating a new supplier

In case a specific template was set for a folder, the generic template was used instead of the folder template when creating a new supplier. This problem is now fixed.

Console crash from the "Index Setup" menu

A console crash could occur from the "Index Setup/settings" menu when validating an index without going to the transcoding tab. This problem is now fixed.

Problem of searching index values following a validation from the index setting interface

When validating an index from the "Index settings" menu, the search parameter (Marker, Value) was changed to a wrong value ( left and right search instead of right and bottom). As a result, when Open Bee Scan Capture found a marker and tried to associate it with a value, he only took into account the values to his left or to his right. This problem is now fixed.

Display refresh problem when adding / removing / modifying an index

Problem when generating the preview of certain pages of documents

(randomly)

File absorption problem causing Open Bee™ Portal slowness

Problem to select all files by using the "Check all" checkbox

Problem with visualization of the type of indexes starting with the sign "-"

Narrowed area at sign height "-"

Problem with dates for which August is written with a circumflex accent

Offset at print from Capture

Popup "Do you want to save changes on the index" that appears all the time

Refresh issue on index management page

When deleting or adding an index, the tables in the page do not refresh with the latest values from the database. This problem is now resolved.

Preview generation problem

When Open Bee Scan Capture absorbs a hundred-page document, not all previews could be generated. This problem is now resolved.

Open Bee Scan Capture can no longer absorb documents from a Windows folder

It happens that Open Bee Scan Capture can no longer absorb documents because its different file observers are turning in a loop. This problem is now resolved

Can not disable VAT rate index

When the VAT rate index is deactivated, the Open Bee Scan Capture engine can no longer detect the values of the different amounts. This problem is now resolved.

Impossibility of interpreting documents that are too large

When a document of more than 150 pages is absorbed by Open Bee Scan Capture, it saturates the resources of the server. This problem is now resolved.

Difficulty detecting a date from a bad text detection

When the text is badly recognized, spaces pollute the detection and prevent the recognition of a valid date. The engine has been improved to take into account this case.

The engine include indexes that should be ignored

When an index is deactivated, it continues to be interpreted by the search engine. Open Bee Scan Capture. This problem has been solved

A change on an index, does not change the detected zone/area

When a user changes the value of an index, the detection zone/area offered to the user does not disappear. The visual has been changed to solve this problem.

Accounting

Pre-assigned account grid is not automatically placed

When an account is filled with a pre-built analytical breakdown grid, it does not automatically apply when it is entered. This problem is now resolved.

Validation problem on a multi-sectional analytical breakdown

There are two types of analytical breakdown in Open Bee Scan Capture. If we manage an analytical breakdown without plan but multisection it can not be validated.

Deleting a third party results in the deletion of the associated account

An account associated with a third party is deleted at the same time as this one, even if this account is used by another third party. This malfunction has been corrected.

Configuration of the accounting import not taken into account

Changing the configuration of the accounting import by text file is taken into account only after clicking save. The changes are now directly applied to the new import without a saving requirement.

Bad/wrong management of a large analytic plan

Accounts that have a large number of accounting plan/analytical sections lead to validation problems. This problem is now fixed.

Inverse negative amounts in accounting entries

A negative amount in the accounting entries table must be placed in the opposite debit / credit column if negative values are not allowed in the accounting entries.

Can not export batch with too many entries.

Batches containing more than 500 rows of accounting entries can not be exported, regardless of the number of entries selected. Prevalidation has been put in place so as not to exceed the authorized limit.
