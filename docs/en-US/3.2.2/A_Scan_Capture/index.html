Ameliorations Open Bee Scan Capture

Restrict documents processed to extensions PDF / TIFF / TIF / PNG / DOC / DOCX

Only PDF / TIFF / TIF / PNG / DOC / DOCX extensions are authorized/allowed to be processed by Open Bee Scan Capture. All other extensions are automatically moved to the folder of the documents in error.

Ability to use the document processing date as metadata sent to Open Bee™ Portal

The date of document processing by Open Bee Scan Capture can now be used as metadata sent to Open Bee™ Portal.

Deleting a regex from the customization interface

It is now possible to delete a regex from the customization interface (from the video coding screen).

Showing the index name on customized zones

The index name is now displayed on the customized zones for better identification.

Ability to set a default list value on an index

It is now possible to set a default value for list type indexes.

Option to replace forbidden characters for outputs (Open Bee™ Portal, Windows)

It is now possible to automatically replace forbidden characters for outputs (Open Bee™ Portal, Windows).

Mongo DB update from 3.2 to 3.4

The Mongo DB database has been updated from 3.2 to 3.4

Possibility of changing the regex of date type indexes

It is now possible to change the regex of date type indexes.

Improved keyword engine for keywords containing spaces

The Keyword Engine used to identify a supplier using a keyword has been improved for keywords containing spaces

Ability to access the index settings directly from the videocoding interface

It is now possible to access the index settings directly from the videocoding interface by double clicking on the name of the index.

Improved file absorption system

The system for absorbing files has been improved to avoid errors in the case where the files are used by another process. The files are now moved to a subfolder named "Files in Progress".

Validation rule

A validation rule can be added to an index so that the user is notified from the video coding interface when the value is not correct.

Using regex in the transcoding conditions of an index

It is now possible to use regex in the transcoding conditions of an index.

Warning the user if a regex is not syntactically correct

A visual warning is displayed in the regex configuration interface if the syntax of a regex entered by the user is not correct.

Ability to completely reset a third-party template

The functionality to reset a template's customizations has been replaced by a full reset of the template to start from the default template.

Addition of a loading indicator when editing an index

It is now possible to follow the progress of the modification of an index on all the templates contained in Open Bee Scan Capture.

Optimizing the reinterpretation of a document

If the template has not changed, the document is no longer reinterpreted unnecessarily

An interface added allowing to manage the different document templates in Open Bee Scan Capture

A new interface makes it possible to manage the different document templates.

Optimization of the overall performance of the Open Bee Scan Capture console

The display performance of the console has been improved, which allows a faster loading of information especially in the case of a large volume of documents.

Addition of a separator field for decimal numbers

Ability to force the separator that must be present on the documents to consider the text in decimal value.

Lock/block validation if a document is detected in duplicate

It is now possible to configure/set Open Bee Scan Capture to block the validation of documents detected as a duplicate.

Ability to choose the number of decimals wanted

It is now possible to choose the number of decimals to display in Open Bee Scan Capture.

Which makes it possible to manage for example the currencies : - without decimals (Africa and the Pacific) - with 3 decimals (Tunisia)

Addition of a constant %OTHER% in the transcoding table

There are now two constants in the transcoding table: %ALL% to manage the accounting setting on all possible values and %OTHER% to manage the setting on all non-parameterized lines.

Accounting

Division of an amount in percentage and registration in the entry template

It is now possible to divide a percentage amount in the accounting entry table and that this division is registered in the entry template.

Possibility of setting the analytical gap

The accounting configuration/setting interface now allows you to choose whether the accounting gap should be: denied, allowed, automatically balanced

Importing analytics in file format

It is now possible to import an analytical plan from a file format.

Third-party account import without centralizer

It is now possible via the import file to import third-party accounts that do not have a default centralizer.

Fusion of two automatic breakdowns on different plans

When two analytical breakdowns come from an automatic generation, they can be merged if they belong to two different plans.

Ability to merge automatically generated accounting entries

It is now possible to merge automatically generated accounting entries for those with identical accounts.

Improved search of analytical sections

It is now possible to search an analytical section by the beginning or the middle of its name or code.

Blocking validation of an entry if a mandatory analytical accounting plan is not completed

From now on, if an anlysis plan is obligatory, the validation of the invoice will be blocked if this accounting plan has not been ventilated.
